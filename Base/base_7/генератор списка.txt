Для какой задачи не подходит использование генератора списка:

1) создание многомерного списка на основе одномерного
2) фильтрация списка по другому списку
3) создание списка на основе словаря
4) создание одномерного списка на основе многомерного
5) последовательное вычисление на основе предыдущих результатов

Давайте разберемся, для какой задачи генераторы списков точны меньше или совсем не точны:

создание многомерного списка на основе одномерного - Генератор списка подходит для этой задачи. Можно использовать вложенные генераторы списков для создания многомерных структур.

фильтрация списка по второму списку — Генератор списка подходит для этой задачи. Можно использовать условные выражения ( if) внутри генератора для фильтрации элементов на основе проверки их международного списка в другом списке.

создание таблицы на основе словаря - Генератор списка подходит для этой задачи. Можно итерироваться по элементам словаря (переключателям, значениям или парам ключ-значение) и создавать список.

создание одномерного списка на основе многомерного - Генератор списка подходит для этой задачи. Можно использовать вложенные циклы forв генераторе списка, чтобы «сгладить» многомерный список в одномерный.

последовательное вычисление на основе предыдущих результатов - Генератор списка не очень подходит для этой задачи. Генераторы списков лучше всего подходят для создания списков, где каждый элемент формируется независимо. Если вам нужно, чтобы вычисление текущего элемента зависело от значения формы элемента, оно становится более сложным и менее понятным. В таких случаях обычно лучше использовать обычный цикл forили другие подходы.
